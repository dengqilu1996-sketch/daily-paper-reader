## 摘要
我们描述了为 SoccerNet 2026 以球员为中心的球动作检测挑战赛设计的系统，该挑战要求预测谁在何时执行何种动作，涵盖广播足球比赛中的八个类别。基于三个 FOOTPASS 基线[1]（TAAD、TAAD+GNN 和 TAAD+DST），我们贡献了四个扩展：(1) 梯度检查点技术，以在单个 GPU 上实现全骨干微调；(2) 将 GNN logit 融合到 DST 编码器中，结合基于图的战术上下文与每个球员的视觉特征；(3) 平方根频率类加权，以解决训练数据中 213:1 的传球与抢断不平衡问题；(4) 一个后处理流程，包括每类 logit 门控、时间帧细化、球衣重新分配以及双模型集成。我们的系统在测试集上取得了 0.548 的 Macro F1，在挑战集上取得了 0.446（服务器评估）。

## Abstract
We describe our system for the SoccerNet 2026 Player-Centric Ball-Action Spotting Challenge, which requires predicting who performs which action and when, across eight classes in broadcast soccer. Building on the three FOOTPASS baselines [1] (TAAD, TAAD+GNN, and TAAD+DST), we contribute four extensions: (1) gradient check pointing to enable full-backbone fine-tuning on a single GPU; (2) fusion of GNN logits into the DST encoder, combining graph-based tactical context with per-player visual features; (3) square-root frequency class weighting to address the 213:1 pass-to-tackle imbalance in the training data; and (4) a post processing pipeline comprising per-class logit gating, temporal frame refinement, jersey re-assignment, and a two-model ensemble. Our system achieves 0.548 Macro F1 on the test set and 0.446 on the challenge set (server evaluation).

### 2. 论文提出的方法论
核心贡献为对 FOOTPASS 基线的四项扩展：

- **梯度检查点微调全主干**
  - 基线 TAAD 因显存限制（22 GB）冻结 X3D 骨干网络。方法增加 X3D 块 0–4 的梯度检查点，允许在 batch size=6 时全主干微调。

- **GNN logits 融合至 DST 编码器**
  - 基线 TAAD+DST 仅将 TAAD 的逐球员 logits 作为序列源。本方法将 TAAD+GNN 产生的 234 维（26 球员 ×9 类）logits 与 TAAD 的 364 维拼接，得到 598 维 token 输入 DST 解码器。
  - 其效果：让解码器同时关注纯视觉特征与图形上下文（球员邻近关系和角色），测试集 Macro F1 从 0.493 提升至 0.505。

- **平方根频率加权训练**
  - 针对类别不平衡，对动作分类交叉熵损失按 `w_c = 1/√(n_c)` 加权，其中 `n_c` 为类别 c 的训练实例数。抢断获得约 10 倍于带球的提升。
  - 仅对动作头加权（角色和帧头不加权）。反频率加权（约 118 倍）会导致罕见类幻觉和常见类召回崩塌，平方根形式避免了此问题。
  - 训练两个 DST 变体：**GNN‑DST**（融合 + 加权）和 **Base‑DST**（仅 TAAD 输入且无加权），作为集成成员。
